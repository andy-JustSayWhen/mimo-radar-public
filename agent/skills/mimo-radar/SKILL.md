# MiMO-Radar

官网已经生成当前智商、智商趋势、模型推荐和智商排行榜。严格按下面三步执行：抓取官网数据、整理官网数据、输出结果。不要重新计算智商或趋势。

## Step 1：抓取官网数据

读取：

`http://193.112.70.227/current.json`

读取成功后进入 Step 2；读取失败时说明官网数据读取失败并结束。

## Step 2：整理官网数据

从返回数据中直接取值：

- 当前智商：`currentIq`
- 当前智商时间：`generatedAt`
- 当前最佳组合：`bestWith`
- 保质量推荐：`qualityRecommendation`
- 性价比推荐：`valueRecommendation`
- 智商趋势：`bestWith.trend.dayOverDay` 和 `bestWith.trend.previousRun`
- 智商排行榜：与“一键接入”页面一致；优先读取 `comboRunSummaries` 中 `totalScore` 不为空的项目，数组为空时读取 `scoreSummaries`，再分别按 `totalScore` 从高到低取 `mimo-v2.5-pro` 前三项和 `mimo-v2.5` 前三项

只整理和格式化这些现成数据，不根据历史数据二次计算，不修改官网给出的结论。

组合统一显示为：

`模型名称［推理开关，计费方式，区域］`

字段转换：

- `on` → `推理-开`
- `off` → `推理-关`
- `token_plan` → `Plan`
- `payg` → `按量`
- `cn` → `中国`
- `intl` → `国际`

显示规则：

- 智商四舍五入为整数。
- `generatedAt` 按北京时间显示为 `MM/DD | HH:MM`。
- `costUsd` 按 `1 USD = ¥7.2` 换算为人民币，保留一位小数；缺失时显示 `成本暂无`。
- `latencyMs` 换算为整数秒；缺失时显示 `耗时暂无`。
- 趋势状态转换：`up` → `智商明显上涨`，`stable` → `智商表现稳定`，`downgraded` → `智商疑似波动`，`unknown` → `数据缺失`。

整理完成后进入 Step 3。
